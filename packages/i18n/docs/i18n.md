# i18n

## Overview

Singleton i18next instance used by the app. Configured with English resources, browser language detection (localStorage + cookie), fallback language, and no interpolation escaping (React handles XSS). Created via `createInstance` and `i18next-browser-languagedetector`.

## Exports

### `resources`

Object with `[defaultLng]: { [defaultNS]: en }` so the default namespace is preloaded with English.

### `i18n` (default export)

The i18next instance:

- `fallbackLng`: `defaultLng` (en)
- `interpolation.escapeValue`: false
- Detector: `lookupLocalStorage` and `lookupCookie` set to `i18nLocalStorageKey` / `i18nCookieKey`, caches: `["localStorage", "cookie"]`
- `init()` is called on the instance

Use `i18n.changeLanguage(lng)`, `i18n.language`, `i18n.addResourceBundle()`, `i18n.hasResourceBundle()`, etc., as per i18next API.

## Usage example

```typescript
import i18n from "@orderly.network/i18n";

i18n.changeLanguage("zh");
console.log(i18n.language);
```
